# Clash 智能切换组 一键配置工具

> 一条命令，让你的 Clash 代理自动故障转移 —— 再也不用手动切节点了。

## 它能做什么？

你有几个常用的代理组（比如「日本自动」「日本手动」「日本故转」），但每次某个挂了都要手动切换？

这个工具帮你创建一个 **fallback（故障转移）组**：
- 优先用第一个代理组
- 每 N 秒自动检查健康状态
- 挂了 → 自动切到下一个
- 下一个也挂了 → 继续切
- 恢复了 → 自动切回来

**全程零手动干预。**

## 快速开始

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/anthropics-inc/clash-smart-group/main/setup.sh)
```

或者下载到本地运行：

```bash
git clone https://github.com/anthropics-inc/clash-smart-group.git
cd clash-smart-group
bash setup.sh
```

然后按提示操作：选客户端 → 选节点 → 设间隔 → 确认 → 搞定。

## 亮点

| 特性 | 说明 |
|------|------|
| **跨平台** | macOS / Linux / Windows (Git Bash / WSL) |
| **多客户端** | FlClash、ClashX Pro、Clash Verge Rev、Clash Verge、Clash for Windows、mihomo |

## 安全机制

```
用户确认 → 备份原文件 → 内存中修改 → 写入文件 → 自动验证（5项）
                                                       │
                                         ┌─────────────┤
                                         │             │
                                      验证通过      验证失败
                                         │             │
                                    显示成功     自动回滚备份
                                                + 显示失败原因
```

**验证项：**
1. YAML 语法合法性（有 PyYAML 时严格解析，无则正则校验）
2. 新 fallback 组存在且 type / interval / proxies 正确
3. 新组在主选择组的第一位（如选择加入）
4. 所有原有组完好无损
5. 重复运行无副作用

**你的配置不会因为工具出错而损坏。**

## fallback 是什么？

Clash 内核原生支持的代理组类型：

| 类型 | 行为 | 适合场景 |
|------|------|---------|
| `select` | 手动选择 | 固定用某个节点 |
| `url-test` | 自动选延迟最低的 | 追求速度 |
| **`fallback`** | **按优先级选第一个可用的** | **稳定性优先、自动故障转移** |

## 参数说明

| 参数 | 作用 | 默认值 |
|------|------|--------|
| 检查间隔 | 多少秒检测一次节点健康 | 20 秒 |
| 超时 | 多久没响应判定为不可用 | 5 秒 |
| lazy | 空闲时是否停止检查 | false（持续检查） |
| url | 检查用的 URL | `https://www.gstatic.com/generate_204` |

## 常见问题

**Q: 更新订阅后配置会丢失吗？**
A: 会。更新订阅会覆盖配置文件，需要重新运行脚本。好消息是只需要 30 秒。

**Q: 对流量有影响吗？**
A: 极小。每次检查约 600 字节（空响应），20 秒间隔 + 3 个组 = 每天约 7.5 MB。

**Q: 支持哪些系统？**
A: macOS / Linux / Windows (需要 Git Bash 或 WSL)。只要有 Python 3 和 Clash 系客户端就行。

## 环境要求

- 任意 Clash 系客户端
